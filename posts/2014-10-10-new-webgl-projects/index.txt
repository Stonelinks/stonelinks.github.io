9:T665,<p>I've been working on some new WebGL related projects for a 3d viewer and editor that targets robotics applications.</p>
<h3><a href="https://github.com/Stonelinks/three.js">My three.js fork</a></h3>
<p>All these projects are built on top of three.js. My fork added kinematics support to the collada loader. <a href="https://stonelinks.github.io/three.js/examples/#webgl_loader_collada_kinematics">Check it out!</a></p>
<h3><a href="https://github.com/Stonelinks/marionette-three.js">Marionette-three.js</a></h3>
<p>This is a library that wraps three.js components for Marionette. Provides a comfortable, event driven interface that abstracts away a lot of three.js things in an MVC friendly way. For example, a <code>Drawable</code> is a <code>Backbone.Model</code> instance that maintains its own <code>three.js</code> scenegraph node. Pass a <code>Backbone.Collection</code> of drawable models to a <code>three.jsRenderer</code> view and you're off to the races! The library also includes support for controls and loading different types of meshes inside <code>Drawable</code> instances. Check out an example <a href="http://stonelinks.github.io/marionette-three.js/example/index.html">here</a> and code for it <a href="https://github.com/Stonelinks/marionette-three.js/tree/master/example">here</a>.</p>
<h3><a href="http://xaphoon.herokuapp.com/">Xaphoon</a></h3>
<p>This is a small heroku app that explores collaborative editing of a 3D environment.
